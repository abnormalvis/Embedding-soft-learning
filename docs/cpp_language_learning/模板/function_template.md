## 函数模板是什么

函数模板是“生成函数的蓝图”，在编译期按实参类型实例化出具体函数。

它适用于：

- 相同逻辑、不同类型
- 希望减少重复代码
- 保持类型安全

---

## 基础语法

```cpp
template <typename T>
T add(T a, T b) {
    return a + b;
}
```

typename 和 class 在模板参数位置等价。

---

## 类型推导与显式指定

```cpp
auto x = add(1, 2);      // T 推导为 int
auto y = add(1.2, 3.4);  // T 推导为 double
```

当推导失败或你想强制类型时，显式指定：

```cpp
auto z = add<long long>(1, 2);
```

## 非类型模板参数

```cpp
template <int N>
void print_times(const std::string& s) {
    for (int i = 0; i < N; ++i) std::cout << s << "\n";
}
```

N 是编译期常量。C++20 还支持更丰富的非类型模板参数场景。

---

## 模板特化与重载关系

优先级上，普通函数重载通常比模板更优先匹配。

```cpp
void f(int);
template <typename T> void f(T);
```

调用 f(10) 时通常选普通函数。

函数模板可以全特化，但复杂场景更建议使用重载或 if constexpr 组织逻辑。

---

## 常见错误

1. 多参数推导冲突导致无法实例化。
2. 过度特化使接口可读性变差。
3. 模板定义放在 cpp 导致链接错误（通常应放头文件）。
4. 错误信息太长时不做最小复现，难以定位。
