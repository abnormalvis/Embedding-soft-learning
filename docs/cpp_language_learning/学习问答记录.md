## Q1: auto 关键字的作用

**问题：** auto 关键字有什么作用？它能用在哪些场景？

**回答：**
`auto` 可以在变量初始化时自动推导类型，实现类型声明的简化。

```cpp
auto i = 10;                          // 推导为 int
auto p = std::make_unique<int>(42);   // 推导为 unique_ptr<int>
auto lambda = [](int x) { return x; }; // 推导为 lambda 类型
```

## Q2: lambda 表达式

**问题：** lambda 的基本语法结构，捕获列表有哪些捕获方式？

**回答：**
lambda 语法：`[捕获列表](参数列表) -> 返回值类型 { 函数体 }`

**捕获方式：**

| 捕获方式 | 语法 | 说明 |
|---------|------|------|
| 不捕获 | `[]` | 什么都不能访问 |
| 按值捕获 | `[=]` | 复制所有外部变量，默认 const |
| 按引用捕获 | `[&]` | 引用外部变量，修改会影响外部 |
| 捕获 this | `[this]` | 捕获当前对象的 this 指针 |
| 混合捕获 | `[=, &x]` | x 按引用，其他按值 |
| 混合捕获 | `[&, x]` | x 按值，其他按引用 |

```cpp
int a = 10;
auto lambda1 = [=]() { return a; };      // 按值捕获a
auto lambda2 = [&]() { return a; };      // 按引用捕获a
auto lambda3 = [=, &a]() { a = 100; };   // a按值，修改引用

// 按值捕获默认是 const，要修改需要 mutable
auto lambda4 = [=]() mutable { a = 20; }; // 修改副本，不影响外部
```

## Q3: std::move 和移动语义

**问题：** 什么是移动语义？为什么需要移动语义？std::move 有什么作用？

**回答：**
**移动语义：** 转移资源所有权，而不是复制数据，避免深拷贝开销。

**移动语义的优势：**
- 不需要分配新内存
- 不需要复制数据
- 移动后原对象变成"空壳"（有效但未定义状态）

```cpp
std::string s1 = "hello";
std::string s2 = std::move(s1);  // 移动后 s1 变成空字符串
```

**std::move 的本质：**
`std::move` 本身不移动任何东西，它只是一个类型转换，将变量转换为右值引用 (`T&&`)，告诉编译器"我可以被移动"。真正执行移动的是移动构造函数或移动赋值运算符。

## Q4: std::unique_ptr

**问题：** 如果对 unique_ptr 调用 std::move 会发生什么？

**回答：**
`std::unique_ptr` 只能移动，不能拷贝。必须用 `std::move` 来转移所有权。

```cpp
auto p1 = std::make_unique<int>(42);
auto p2 = std::move(p1);  // ✅ 完全合法

// 移动后
// p2 拥有资源（值为 42）
// p1 变成空（值为 nullptr）
```

## Q5: std::shared_ptr 和 std::weak_ptr

**问题：** shared_ptr 和 weak_ptr 的区别和应用场景？

**回答：**
**shared_ptr：** 共享所有权，有引用计数。可被多个对象持有同一资源。

**weak_ptr：** 不参与引用计数，不拥有资源。主要用于：
1. 打破 shared_ptr 循环引用
2. 临时观察资源，不影响生命周期

```cpp
// 循环引用会导致内存泄漏
class A { std::shared_ptr<B> b; };
class B { std::shared_ptr<A> a; };  // ❌ 循环引用

// 用 weak_ptr 解决
class B { std::weak_ptr<A> a; };  // ✅ 打破循环

// 使用 weak_ptr 前必须 lock()
std::weak_ptr<int> wp = std::make_shared<int>(42);
if (auto sp = wp.lock()) {
    cout << *sp << endl;  // 安全使用
}
```

**应用场景：**
- shared_ptr：多个对象操作同一个资源，如多个控制器共享同一个关节句柄
- weak_ptr：观察者模式、缓存、避免循环引用

## Q6: RAII

**问题：** RAII 是什么？它和资源管理有什么关系？

**回答：**
RAII = "Resource Acquisition Is Initialization"，即**资源获取即初始化**。

核心思想：资源的获取和释放与对象的生命周期绑定。

```cpp
class FileHandler {
    std::fstream file;
public:
    FileHandler(const string& name) {
        file.open(name);  // 获取资源（打开文件）
    }
    ~FileHandler() {
        file.close();  // 释放资源（自动调用）
    }
};

void func() {
    FileHandler fh("data.txt");  // 构造函数获取资源
    // ... 使用文件 ...
}  // 析构函数自动释放资源，不用手动 close!
```

**优势：**
- 自动释放资源，避免内存泄漏
- 异常安全（即使发生异常，析构函数也会执行）
- 代码更简洁，不需要手动释放

## Q7: std::lock_guard

**问题：** std::lock_guard 是如何利用 RAII 实现自动释放锁的？

**回答：**
```cpp
std::mutex mtx;

void func() {
    std::lock_guard<std::mutex> lock(mtx);  // 构造函数获取锁
    // ... 临界区代码 ...
}  // 析构函数自动释放锁！
```

**原理：**
- 构造函数中调用 `mtx.lock()` 获取锁
- 析构函数中调用 `mtx.unlock()` 释放锁
- 无需手动 unlock，即使代码抛出异常，析构函数也会执行

## Q8: 虚函数和多态

**问题：** 虚函数和多态是如何实现的？

**回答：**
C++ 通过**虚函数表 (vtable)** 实现多态：

```cpp
class Animal {
public:
    virtual void speak() { cout << "some sound" << endl; }
    virtual ~Animal() {}
};

class Dog : public Animal {
public:
    void speak() override { cout << "wang!" << endl; }
};

Animal* animal = new Dog();
animal->speak();  // 输出 "wang!"，不是 "some sound"
```

**工作原理：**
- 每个包含虚函数的类有一个虚函数表（函数指针数组）
- 每个对象有一个虚指针 (vptr) 指向这个表
- 调用虚函数时，通过 vptr 查表，找到实际要调用的函数

**注意：** 基类析构函数应为虚函数，否则删除派生类指针时只调用基类析构函数。

## Q9: 纯虚函数和抽象类

**问题：** 什么是纯虚函数和抽象类？

**回答：**
**纯虚函数：** 在基类中声明但不实现的虚函数，使用 `= 0` 语法。

```cpp
class Animal {
public:
    virtual void speak() = 0;  // 纯虚函数
    virtual ~Animal() {}
};
```

**抽象类：** 包含纯虚函数的类，不能实例化，只能作为基类。

```cpp
Animal a;          // ❌ 错误！抽象类不能实例化
Animal* a;         // ✅ 可以定义指针
```

子类必须实现所有纯虚函数才能实例化。

**作用：** 定义接口，规范子类的行为，强制子类实现特定方法。

## Q11: std::vector 工作原理

**问题：** vector 的扩容机制和迭代器失效问题？

**回答：**
**扩容机制：**
- vector 使用堆内存
- 当容量不够时，分配更大的内存（通常是 2 倍）
- 复制所有元素到新内存，释放旧内存

**迭代器失效：**

```cpp
vector<int> v = {1, 2, 3, 4, 5};
auto it = v.begin();  // 指向元素 1

v.push_back(6);       // 触发扩容！整个内存可能移动
// 原来的 it 失效了！

cout << *it;          // ❌ 未定义行为！可能崩溃
```

**安全遍历并添加元素：**

```cpp
// 方法1：使用索引
for (size_t i = 0; i < v.size(); ++i) {
    if (v[i] == 2) {
        v.push_back(99);  // 索引方式安全
    }
}

// 方法2：使用 erase-remove idiom
v.erase(
    std::remove_if(v.begin(), v.end(), [](int x) { return x % 2 == 0; }),
    v.end()
);
```

## Q12: STL 容器分类

**问题：** 顺序容器、关联容器、无序容器的区别和适用场景？

**回答：**

| 分类 | 容器 | 特点 | 适用场景 |
|------|------|------|----------|
| **顺序容器** | vector | 随机访问快，尾部插入快 | 频繁随机访问，尾部添加 |

```cpp
std::map<string, int> scores;        // 需要排序

```

**map vs unordered_map：**
- map：红黑树，O(log n)，有序
- unordered_map：哈希表，O(1)，无序

## Q13: 函数指针和函数对象

**问题：** 函数指针和函数对象的区别？

**回答：**

| | 函数指针 | 函数对象（Functor） |
|---|---|---|
| 灵活性 | 较低 | 高 |
| 内联优化 | 通常不行 | 可以 |
| 状态保存 | 依赖全局/静态变量 | 通过成员变量 |
| 类型安全 | 较弱 | 强 |

```cpp

class Adder {
    int bias;
public:
    Adder(int b) : bias(b) {}
    int operator()(int x) const { return x + bias; }
};

Adder add10(10);
cout << add10(5);  // 输出 15
```

**lambda 本质：** lambda 是编译器生成的匿名函数对象（functor）。

```cpp
auto lambda = [=](int x) { return x + 1; };

// 编译器生成类似这样的类
class __lambda {
public:
    int operator()(int x) const { return x + 1; }
};
```

## Q15: 数据竞争和同步机制

**问题：** 数据竞争是如何产生的？有哪些同步机制？

**回答：**
**数据竞争：** 两个或以上的线程访问同一块内存或资源，且至少有一个是写操作。

**同步机制：**

| 机制 | 作用 |
|------|------|
| std::mutex | 互斥锁，保护共享数据 |
| std::lock_guard | RAII 锁，自动释放 |
| std::unique_lock | 灵活锁，支持延迟锁、条件变量 |
| std::condition_variable | 条件变量，线程间协作 |
| std::atomic | 原子操作，无锁计数器 |

**mutex vs atomic：**

| | std::mutex | std::atomic |
|---|---|---|
| 原理 | 内核态互斥锁，可能阻塞 | 硬件级原子操作 |
| 性能 | 较重，涉及系统调用 | 轻量，极快 |
| 适用场景 | 复杂保护、多行代码 | 简单值（计数器、标志） |

## Q16: std::condition_variable

**问题：** condition_variable 用法？虚假唤醒问题？

**回答：**
```cpp
std::mutex mtx;
std::condition_variable cv;
bool ready = false;

void worker() {
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, []{ return ready; });  // 等待 ready 为 true
    cout << "工作开始" << endl;
}

void notifier() {
    {
        std::lock_guard<std::mutex> lock(mtx);
        ready = true;
    }
    cv.notify_one();  // 通知一个等待的线程
}
```

**谓词的作用：** 防止虚假唤醒。线程可能因系统调度等原因从 wait 返回，但实际条件并未满足。

```cpp
// 没有谓词（危险！）
cv.wait(lock);  // 醒来后不知道条件是否真的满足

// 有谓词（安全！）
cv.wait(lock, []{ return ready; });  // 醒来后自动检查条件
```

## Q17: 异常处理和异常安全

**问题：** 异常处理机制？RAII 在异常安全中的重要性？

**回答：**
```cpp
try {
    int age = -1;
    if (age < 0) {
        throw std::invalid_argument("年龄不能为负数");
    }
} catch (const std::exception& e) {
    cout << "捕获异常: " << e.what() << endl;
}
```

**RAII 在异常安全中的重要性：**

```cpp
// 没有 RAII：异常导致锁泄漏
void func() {
    mtx.lock();
    do_something_that_may_throw();  // 异常！
    mtx.unlock();  // 永远不会执行
}

// 使用 RAII：自动释放
void func_safe() {
    std::lock_guard<std::mutex> lock(mtx);
    do_something_that_may_throw();  // 即使异常，析构会释放锁
}
```

## Q18: constexpr vs consteval

**问题：** constexpr 和 consteval 的区别？

**回答：**

| | constexpr | consteval |
|---|---|---|
| 执行时机 | 编译时或运行时（视参数而定） | 必须编译时 |
| 参数要求 | 可以接受常量或变量 | 只能接受常量 |
| 不满足时 | 退化为运行时 | 编译错误 |
| 可修饰 | 变量和函数 | 只能修饰函数 |

```cpp
constexpr int add(int x) { return x + 1; }
consteval int mul(int x) { return x * 2; }

int a = 10;
int r2 = add(a);             // ✅ 运行时执行（退化）
int r4 = mul(a);             // ❌ 编译错误！参数不是常量
```

**应用场景：**
- consteval：编译时计算、元编程、强制零开销抽象
- constexpr：性能优化、泛型编程

## Q19: 右值引用和完美转发

**问题：** 右值引用和完美转发？

**回答：**
**右值引用 (`T&&`)：**

```cpp
void foo(int&& x) { }  // 只能接收右值
foo(10);  // ✅

template<typename T>
void bar(T&& x) { }  // 万能引用，既能接收左值也能接收右值
bar(10);       // T=int，x 是 int&&
bar(x);        // T=int&，x 是 int&
```

**完美转发 (`std::forward`)：** 保持原始参数的左值/右值属性。

```cpp
template<typename T>
void wrapper(T&& arg) {
    process(std::forward<T>(arg));  // 保持原始属性
}
```

**引用折叠规则：**

| 原始类型 | 折叠后 |
|----------|--------|
| T& & | T& |
| T& && | T& |
| T&& & | T& |
| T&& && | T&& |

## Q24: new/delete vs malloc/free

**问题：** new/delete 和 malloc/free 的区别？placement new 是什么？

**回答：**

| | new/delete | malloc/free |
|---|---|---|
| 内存区域 | 堆 | 堆 |
| 类型安全 | 类型安全 | void*，需手动转换 |
| 构造函数 | new 会调用 | 不调用 |
| 析构函数 | delete 会调用 | 不调用 |

```cpp
int* p1 = new int(10);    // 分配 + 构造
delete p1;

int* p2 = (int*)malloc(sizeof(int));  // 只分配
free(p2);
```

**数组释放：**
```cpp
int* arr = new int[10];
delete[] arr;  // 用 delete[] 释放数组
```

**placement new：** 在已分配的内存上构造对象，不分配新内存。

```cpp
char buffer[sizeof(Widget)];
Widget* w = new (buffer) Widget();  // 在 buffer 上构造
w->~Widget();                        // 手动调用析构
```

## Q27: 类型别名和类型推断

**问题：** using vs typedef？decltype 和 declval？

**回答：**
```cpp
// using 现代风格
using MapType = std::map<std::string, int>;

// using 支持模板
template<typename T>
using Ptr = std::unique_ptr<T>;

// decltype 提取类型
std::vector<int> v;
decltype(v)::value_type x;

// declval 不创建对象就能获取类型
std::decay_t<decltype(std::declval<T>() + std::declval<U>())>
```

**std::decay：** 先去掉引用，然后去掉 const/volatile，把数组退化为指针，把函数退化为函数指针。比 `std::remove_reference` 更强大。

## Q28: SOLID 原则

**问题：** SOLID 原则在 C++ 中如何应用？

**回答：**

| 原则 | 说明 | C++ 实现 |
|------|------|----------|
| SRP | 单一职责 | 每个类只做一件事 |
| OCP | 开闭原则 | 基类定义虚函数，子类扩展 |
| LSP | 里氏替换 | 子类必须能替换基类 |
| ISP | 接口隔离 | 拆分大接口为多个小接口 |
| DIP | 依赖反转 | 依赖抽象基类，不依赖具体实现 |

**里氏替换原则 (LSP)：**
子类对象可以替换父类对象，而不破坏程序正确性。

```cpp
// 违反 LSP：子类不能做父类的行为
class Bird {
    virtual void fly() = 0;
};
class Penguin : public Bird {
    void fly() override { throw "企鹅不会飞！"; }
};
```

**依赖倒置示例：**
```cpp
class IDatabase {
    virtual void save(const Data&) = 0;
    virtual Data load() = 0;
};

class MySQLDB : public IDatabase { };
class RedisCache : public IDatabase { };

class Service {
    std::unique_ptr<IDatabase> db_;  // 依赖抽象
};
```

## Q34: C++20 新特性

**问题：** C++20 重要特性？concepts、ranges 是什么？

**回答：**
**concepts：** 约束模板参数类型，编译时检查更清晰。

```cpp
template<typename T>
concept Numeric = std::integral<T> || std::floating_point<T>;

template<Numeric T>
T add(T a, T b) { return a + b; }

add(1, 2);     // ✅ int 是 Numeric
add(1.0, 2.0); // ✅ double 是 Numeric
add("a", "b"); // ❌ 编译错误
```

**ranges：** 函数式编程风格的容器操作。

```cpp
#include <ranges>
std::vector<int> v{1,2,3,4,5};
auto result = v | std::views::filter([](int x) { return x % 2; })
                 | std::views::transform([](int x) { return x * 2; });
```

**coroutines：** 协程，async/await 支持。

## Q35: 协程 vs 线程

**问题：** 协程和线程的区别？适合什么场景？

**回答：**

| | 协程 (Coroutine) | 线程 (Thread) |
|---|---|---|
| 管理者 | 程序员（用户态） | 操作系统（内核态） |
| 切换开销 | 极低（函数调用级别） | 较高（上下文切换） |
| 数量 | 可以有数百万个 | 受限于系统（几千个） |
| 同步 | 无需锁（协作式） | 需要 mutex 等同步 |
| 用途 | IO 密集型任务 | CPU 密集型任务 |

**适合协程的场景：**
- 高并发网络服务（Web 服务器）
- 异步 IO 操作
- 游戏服务器
- 需要处理大量连接但 CPU 使用不高的场景

## 接口设计

**问题：** 如何设计一个接口？

**回答：**
定义纯虚函数 + 虚析构函数。

```cpp
class IInterface {
public:
    virtual ~IInterface() {}  // 虚析构，确保正确清理
    virtual void doSomething() = 0;  // 纯虚函数
};
```

---

## 设计接口注意事项

1. 包含虚析构函数
2. 纯虚函数定义接口契约
3. 子类必须实现所有纯虚函数才能实例化
