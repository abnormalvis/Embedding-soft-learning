## 基础写法

C++11 起可通过 std::thread 创建线程。线程启动后会立即并发执行传入的可调用对象。

```cpp
#include <iostream>
#include <thread>

void task() {
    std::cout << "hello from worker\n";
}

int main() {
    std::thread t(task);
    t.join();
    return 0;
}
```

编译：

```bash
g++ -std=c++20 -Wall -Wextra -O2 main.cpp -pthread -o app
```

## 可调用对象类型

std::thread 可接受以下类型：

- 普通函数
- lambda 表达式
- 函数对象（仿函数）
- 成员函数 + 对象指针/引用

```cpp
#include <iostream>
#include <thread>

struct Job {
    void operator()(int x) const {
        std::cout << "functor: " << x << "\n";
    }
};

class Worker {
public:
    void run(int x) {
        std::cout << "member: " << x << "\n";
    }
};

int main() {
    std::thread t1([] { std::cout << "lambda\n"; });
    std::thread t2(Job{}, 42);

    Worker w;
    std::thread t3(&Worker::run, &w, 7);

    t1.join();
    t2.join();
    t3.join();
    return 0;
}
```

## 参数传递规则

默认按值拷贝传参。

- 小对象按值通常更安全。
- 需要引用语义时，用 std::ref 或 std::cref。

```cpp
#include <functional>
#include <iostream>
#include <thread>

void add_one(int& n) {
    ++n;
}

int main() {
    int value = 10;
    std::thread t(add_one, std::ref(value));
    t.join();
    std::cout << value << "\n"; // 11
    return 0;
}
```

## join 与 detach

### join

- 当前线程等待目标线程结束。
- 结束后可以安全回收线程资源。

### detach

- 线程与 thread 对象分离，后台独立运行。
- 分离后不可再 join。
- 需要确保被访问对象在后台线程结束前仍有效。

经验规则：能 join 就 join。detach 仅用于明确的后台守护型任务。

## jthread（C++20）

std::jthread 是 C++20 提供的改进线程类：

- 析构时自动请求停止并 join，降低忘记 join 的风险。
- 支持 stop_token 进行协作式停止。

```cpp
#include <chrono>
#include <iostream>
#include <thread>

void loop(std::stop_token st) {
    while (!st.stop_requested()) {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
    std::cout << "stopped\n";
}

int main() {
    std::jthread jt(loop);
    std::this_thread::sleep_for(std::chrono::milliseconds(300));
    // 离开作用域时 jt 自动停止并回收
    return 0;
}
```

## 常见错误

1. 线程对象析构前未 join 或 detach，程序会 terminate。
2. 传引用但没用 std::ref，导致误拷贝。
3. detach 后访问局部变量，触发悬垂引用。
4. 线程函数抛异常未捕获，导致进程终止。
