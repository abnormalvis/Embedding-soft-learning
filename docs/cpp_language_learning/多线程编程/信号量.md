## 信号量是什么

信号量维护一个“许可计数”，用于控制并发访问数量：

- acquire：消耗许可，若无许可则等待。
- release：归还许可，唤醒等待线程。

它适合“限流”，而不只是“互斥”。

---

## C++20 counting_semaphore

```cpp
#include <semaphore>

std::counting_semaphore<3> sem(3); // 最大3，初值3
```

模板参数是最大许可数，构造参数是初始许可数。

## 基本示例

```cpp
#include <chrono>
#include <iostream>
#include <semaphore>
#include <thread>
#include <vector>

std::counting_semaphore<2> sem(2);

void task(int id) {
    sem.acquire();
    std::cout << "task " << id << " enter\n";
    std::this_thread::sleep_for(std::chrono::milliseconds(200));
    std::cout << "task " << id << " leave\n";
    sem.release();
}

int main() {
    std::vector<std::thread> ts;
    for (int i = 0; i < 5; ++i) ts.emplace_back(task, i);
    for (auto& t : ts) t.join();
}
```

## 典型场景

- 数据库连接池并发上限。
- API 请求并发限流。
- 线程池中特定任务类型的并发阈值。

---

## 与 mutex 的区别

- mutex：同一时刻仅 1 个线程进入临界区。
- semaphore：同一时刻允许 N 个线程进入。

如果只是保护单个临界区一致性，优先 mutex；如果是资源配额管理，优先 semaphore。

---

## 常见坑

1. acquire/release 不配对，许可泄漏。
2. 把信号量当条件变量用，语义错位。
3. 许可值设计过大，导致资源打爆。
4. 忽视异常路径，release 未执行。
