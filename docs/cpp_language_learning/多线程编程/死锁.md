## 什么是死锁

死锁是多个线程互相等待对方持有的资源，导致系统无进展。

直接后果：线程永久阻塞，吞吐下降，服务不可用。

---

## 四个必要条件

1. 互斥
2. 请求并保持
3. 不可剥夺
4. 循环等待

四者同时满足才会形成死锁。破坏任意一个条件都可规避。

## 典型死锁示例

```cpp
#include <mutex>
#include <thread>

std::mutex m1, m2;

void t1() {
    std::lock_guard<std::mutex> a(m1);
    std::this_thread::sleep_for(std::chrono::milliseconds(50));
    std::lock_guard<std::mutex> b(m2);
}

void t2() {
    std::lock_guard<std::mutex> a(m2);
    std::this_thread::sleep_for(std::chrono::milliseconds(50));
    std::lock_guard<std::mutex> b(m1);
}
```

改进：统一加锁顺序，或使用 std::scoped_lock。

```cpp
std::scoped_lock lock(m1, m2);
```

---

## 规避策略

- 统一锁顺序（强烈推荐）。
- 缩小临界区，减少持锁时间。
- 使用 scoped_lock 处理多锁。
- try_lock + 超时回退。
- 锁内避免 I/O 与复杂调用。

---

## 排查方法

1. 记录锁获取/释放日志（线程 id + 锁 id）。
2. gdb 查看线程栈，识别阻塞点。
3. 对高风险路径做超时保护，尽早报警。
4. 代码评审时重点检查“多锁顺序是否一致”。
