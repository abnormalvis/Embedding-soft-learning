## 为什么需要线程池

线程池的目标是复用线程，避免频繁创建/销毁线程带来的开销。

收益：

- 降低线程管理成本
- 限制并发上限，保护系统
- 统一任务调度与回收

---

## 线程池核心组件

- 工作线程集合
- 任务队列
- 互斥锁 + 条件变量
- 停止标志
- 任务结果通道（通常是 future）

---

## 最小线程池实现思路

1. 启动 N 个 worker 线程，循环取任务。
2. 提交任务时入队并通知一个 worker。
3. 析构时设置 stop，广播唤醒，join 全部线程。
4. 用 packaged_task 封装任务并返回 future。

## 工程注意事项

1. 任务队列需上限保护，防止内存膨胀。
2. 停机流程必须可控：拒收新任务 + 消费存量任务。
3. 避免任务里长期阻塞，影响线程池吞吐。
4. 异常需被 future 捕获，不能让 worker 崩溃退出。
5. 线程数一般设为 CPU 核心数附近，I/O 型任务可适度放大。
