## 原子操作的意义

原子操作保证“不可分割”：要么完整执行，要么不执行。它可以在不加互斥锁的情况下，安全更新单个共享变量。

典型用途：

- 计数器
- 状态标志（如 stop）
- 轻量统计

## std::atomic 基础

```cpp
#include <atomic>
#include <thread>
#include <iostream>

std::atomic<int> counter{0};

void work() {
    for (int i = 0; i < 100000; ++i) {
        ++counter;
    }
}

int main() {
    std::thread t1(work), t2(work);
    t1.join();
    t2.join();
    std::cout << counter.load() << "\n";
}
```

## 常见原子接口

- load / store
- fetch_add / fetch_sub
- compare_exchange_weak / strong
- exchange

CAS（比较并交换）是无锁算法的核心原语。

```cpp
int expected = 0;
std::atomic<int> x{0};
if (x.compare_exchange_strong(expected, 1)) {
    // x 由 0 改为 1
}
```

## 内存序简述

常见内存序：

- memory_order_relaxed：只保证原子性，不保证顺序。
- memory_order_acquire / release：建立线程间同步关系。
- memory_order_seq_cst：最强、最直观、开销通常更高。

学习建议：先用默认 seq_cst 写对，再做性能优化。

## 什么时候用 atomic

优先 atomic：

- 单变量读写或加减。
- 不需要复合不变量。

不要勉强 atomic：

- 多变量需要一致性。
- 需要阻塞等待条件。

此时改用 mutex 或 condition_variable。

---

## 常见坑

1. 把多个原子变量当作“整体事务”使用。
2. 自旋等待没有退避策略，导致 CPU 飙高。
3. 误用 relaxed 导致可见性和时序错误。
4. 认为 atomic 一定比 mutex 快，忽略场景差异。
