## barrier 用来做什么

barrier 用于“阶段同步”：多个线程都到达同步点后，才能一起进入下一阶段。

它适合并行分阶段任务，如：

- 每轮并行计算后统一汇总
- 多线程迭代算法

---

## 与 latch 的区别

- latch：一次性倒计时门闩，归零后结束。
- barrier：可重复使用，每轮都可同步。

## C++20 基本示例

```cpp
#include <barrier>
#include <chrono>
#include <iostream>
#include <thread>
#include <vector>

void worker(int id, std::barrier<>& b) {
    std::cout << "worker " << id << " phase1\n";
    std::this_thread::sleep_for(std::chrono::milliseconds(100 * id));

    b.arrive_and_wait();

    std::cout << "worker " << id << " phase2\n";
}

int main() {
    constexpr int n = 4;
    std::barrier sync_point(n);
    std::vector<std::thread> ts;

    for (int i = 0; i < n; ++i) ts.emplace_back(worker, i + 1, std::ref(sync_point));
    for (auto& t : ts) t.join();
}
```

## 使用场景

- 并行图像处理（分块计算后统一合并）。
- 数值迭代（每轮迭代后全体对齐）。
- 仿真系统（多子系统按 tick 同步）。

---

## 常见坑

1. 参与线程数量配置错误，导致永久等待。
2. 某线程提前退出，其他线程卡死在 barrier。
3. 把 barrier 当互斥锁使用，语义不匹配。
