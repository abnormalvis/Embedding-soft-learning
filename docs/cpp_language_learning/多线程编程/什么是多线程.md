## 核心概念

线程是 CPU 调度的最小执行单位。一个进程可以包含多个线程，它们共享进程资源，但各自有独立的执行栈。

在 C++11 之前，C++ 标准库没有统一线程模型；从 C++11 开始，标准库提供了跨平台并发组件，例如 thread、mutex、condition_variable、atomic。

---

## 进程与线程

| 维度 | 进程 | 线程 |
| --- | --- | --- |
| 资源拥有 | 独立地址空间与系统资源 | 共享所属进程资源 |
| 创建成本 | 高 | 相对低 |
| 通信方式 | IPC（管道、共享内存等） | 共享内存 + 同步原语 |
| 崩溃影响 | 通常不直接影响其他进程 | 可能导致整个进程异常 |

要点：线程轻量但更容易出现共享数据问题，必须配合同步机制使用。

---

## 并发与并行

- 并发：同一时间段内多个任务交替推进。
- 并行：同一时刻多个任务在不同核心上同时执行。

在单核 CPU 上，线程主要体现并发；在多核 CPU 上，才可能获得并行计算收益。

---

## 为什么要用多线程

- 提高吞吐：把可独立任务拆给多个线程。
- 提升响应：主线程负责交互，耗时任务下放后台。
- 隐藏等待：I/O 阻塞期间，其他线程可继续运行。

注意：不是线程越多越快。线程切换和同步开销会吞掉收益。

---

## 线程生命周期

典型状态可以理解为：

1. 创建：构造 thread 对象，尚未结束。
2. 可运行：等待调度。
3. 运行：获得时间片执行。
4. 阻塞：等待锁、I/O、条件变量等。
5. 结束：函数返回或异常终止。

C++ 编程重点：线程对象生命周期必须显式管理。可 join 就 join，不需要等待才 detach。

## 多线程的风险

### 数据竞争

多个线程无同步地访问同一内存位置，且至少一个是写操作，行为未定义。

### 死锁

线程 A 等待线程 B 的锁，线程 B 反过来等待线程 A 的锁，双方永远卡住。

### 活锁与饥饿

线程看起来在工作但无进展，或长期抢不到资源。

### 内存可见性问题

一个线程写入的数据，另一个线程未必立即可见，需要通过锁或原子语义建立同步关系。

## 最小示例

```cpp
#include <iostream>
#include <thread>

void worker(int id) {
    std::cout << "worker " << id << " running\n";
}

int main() {
    std::thread t1(worker, 1);
    std::thread t2(worker, 2);

    t1.join();
    t2.join();
    return 0;
}
```

编译命令：

```bash
g++ -std=c++20 -Wall -Wextra -O2 main.cpp -pthread -o app
```
