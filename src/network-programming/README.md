# C 语言网络编程综合练习 - 文件传输系统

## 项目简介
这是一个基于 TCP 协议的客户端 - 服务器文件传输系统，实现了通过命令从服务器下载不同类型的文件。

## 功能特性
1. **服务器端**：
   - 启动服务进程，监听端口 9999
   - 接受客户端连接
   - 根据客户端命令发送对应的文件（txt、图片、视频）
   - 支持多个客户端轮流连接

2. **客户端**：
   - 连接到服务器后显示交互式菜单
   - 通过按键输入命令：
     - `1` + 回车：下载 TXT 文本文件
     - `2` + 回车：下载 BMP 图片文件
     - `3` + 回车：下载 MP4 视频文件
     - `q` + 回车：退出程序
   - 实时显示下载进度
   - 自动保存文件到 downloads 目录

## 项目结构
```
network-programming/
├── src/                    # 源代码目录
│   ├── server_demo.c      # 服务器端代码
│   └── client_demo.c      # 客户端代码

│   └── test_video.mp4     # 测试视频文件

├── build/                  # 编译输出目录
│   ├── server_demo        # 服务器可执行文件
│   └── client_demo        # 客户端可执行文件

└── test.sh                # 自动化测试脚本
```

## 编译方法
```bash
cd build
cmake ..
make
```

## 使用方法

### 方法一：手动运行
1. 启动服务器：
   ```bash
   ./build/server_demo
   ```

2. 在另一个终端运行客户端：
   ```bash
   ./build/client_demo
   ```

3. 根据菜单提示输入命令（1、2、3 或 q）

### 方法二：使用测试脚本
```bash
./test.sh
```
测试脚本会自动：
- 检查测试文件
- 启动服务器
- 运行客户端并依次测试所有命令
- 验证下载结果

## 技术要点
1. **TCP 套接字编程**：使用 socket、bind、listen、accept、connect
2. **文件 I/O**：使用 open、read、write、close 进行文件操作
3. **二进制文件传输**：正确处理文本和二进制文件
4. **协议设计**：先发送文件大小，再发送文件内容
5. **错误处理**：完善的错误检查和提示
